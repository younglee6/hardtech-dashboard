# Diagnosis Playbook

## Signal Fusion

对同一问题同时检查三类信号：

1. 行为数据
- 用户路径卡点是否集中在固定步骤
- 关键行为触发频次是否下滑
- 新老用户影响是否对称

2. 运营数据
- 渠道结构是否变化（高质量渠道占比下降）
- 活动曝光、点击、转化是否断层
- 预算、出价、投放素材是否变更

3. 用户反馈
- 低分评价主题是否集中
- 客服工单是否出现高频关键词
- 社媒负面提及是否升高

## Root-Cause Hypothesis Template

每条假设都按以下结构表达：

- 现象：什么指标在什么时间段发生异常
- 假设：可能原因是什么
- 证据：支持与反证各是什么
- 置信度：高/中/低
- 验证动作：如何在 1-2 周内验证

## Prioritization

## ICE

- Impact: 对核心指标潜在提升
- Confidence: 证据强度与可验证性
- Ease: 落地复杂度与资源消耗

优先执行：Impact 高 + Confidence 中高 + Ease 高。

## RICE

- Reach: 影响用户规模
- Impact: 人均影响强度
- Confidence: 结论可信度
- Effort: 人日投入

适用于候选动作较多、需要跨团队排期时。

## 2-Week Execution Rhythm

1. 第 1-2 天：口径确认 + 异常定位
2. 第 3-4 天：根因假设 + 方案评审
3. 第 5-10 天：快速实验上线 + 每日监控
4. 第 11-14 天：复盘与放大/回滚决策
